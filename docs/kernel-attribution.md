# Kernel Attribution — Branding Hyperion in the Source Tree

This document explains exactly how to modify the Linux kernel source so that
all version strings, `uname` output, and boot messages correctly identify as
**Hyperion Kernel by Soumalya Das**.

Expected result:

```
Linux version 6.19.6-Hyperion-0.1.0 (Soumalya Das) (gcc version 13.2.0) #1 SMP PREEMPT 2026
```

---

## 1. Makefile — Top-level version string

Edit `linux-6.19.6/Makefile`:

```makefile
# Original
VERSION = 6
PATCHLEVEL = 12
SUBLEVEL = 0
EXTRAVERSION =
NAME = Hurr durr I'ma ninja sloth

# Hyperion: change NAME and EXTRAVERSION
VERSION = 6
PATCHLEVEL = 12
SUBLEVEL = 0
EXTRAVERSION = -Hyperion-0.1.0
NAME = Hyperion
```

The `EXTRAVERSION` is what appears in `uname -r` **when not overridden by**
`CONFIG_LOCALVERSION`. Since we also set `CONFIG_LOCALVERSION="-Hyperion-0.1.0"`
in `hyperion.config`, they work together. Set `EXTRAVERSION` in Makefile to
empty and rely on `CONFIG_LOCALVERSION` for flexibility, OR set both to the
same value for a hard-coded build.

**Recommended approach (flexible):**

```makefile
EXTRAVERSION =
# CONFIG_LOCALVERSION="-Hyperion-0.1.0" in .config handles the suffix
```

---

## 2. CONFIG_LOCALVERSION — In the .config

```kconfig
# In hyperion.config (already set):
CONFIG_LOCALVERSION="-Hyperion-0.1.0"
CONFIG_LOCALVERSION_AUTO=n
```

This produces: `6.19.6-Hyperion-0.1.0`

**Build-time override** (takes precedence over config):

```bash
make -j$(nproc) LOCALVERSION="-Hyperion-0.1.0"
```

---

## 3. init/version.c — Author credit in version string

This file generates the string printed at boot and available via `uname -v`.

Edit `linux-6.19.6/init/version.c`:

```c
/* Original line: */
const char linux_proc_banner[] =
    "%s version %s"
    " (" linux_compile_by "@" linux_compile_host ")"
    " (" linux_compiler ")\n";

/* Hyperion: replace with hardcoded author string */
const char linux_proc_banner[] =
    "%s version %s"
    " (Soumalya Das)"
    " (" linux_compiler ")\n";
```

Then in the same file, find and update the build info strings:

```c
/* Original: set by environment / kbuild */
const char *linux_compile_by  = CONFIG_BUILD_USER;
const char *linux_compile_host = CONFIG_BUILD_HOST;

/* Hyperion override: */
const char *linux_compile_by  = "Soumalya Das";
const char *linux_compile_host = "hyperion-build";
```

**Simpler approach — use Kconfig build options:**

In your build environment:

```bash
export KBUILD_BUILD_USER="Soumalya Das"
export KBUILD_BUILD_HOST="hyperion-build"
make -j$(nproc) LOCALVERSION="-Hyperion-0.1.0"
```

This is the **recommended approach** as it requires no source modification.
Set these in `scripts/build-kernel.sh`:

```bash
export KBUILD_BUILD_USER="Soumalya Das"
export KBUILD_BUILD_HOST="hyperion-build"
export KBUILD_BUILD_TIMESTAMP="2026"
```

---

## 4. scripts/setlocalversion — Control the version suffix

This script computes the `localversion` suffix from git tags and appends it
to `CONFIG_LOCALVERSION`. Since `CONFIG_LOCALVERSION_AUTO=n`, the auto-detect
is disabled and our static string is used.

If you want to also embed the git commit hash:

```bash
# In scripts/setlocalversion, near the end, append custom string:
echo "-Hyperion-0.1.0-$(git rev-parse --short HEAD 2>/dev/null || echo 'release')"
```

This produces: `6.19.6-Hyperion-0.1.0-a3f9c21`

## 5. include/linux/uts.h — UTS (Unix Timesharing System) strings

```c
/* UTS_SYSNAME is "Linux" — leave this */
#define UTS_SYSNAME "Linux"

/* UTS_NODENAME — hostname (set by CONFIG_DEFAULT_HOSTNAME) */
#define UTS_NODENAME CONFIG_DEFAULT_HOSTNAME   /* "hyperion" */

/* UTS_RELEASE — kernel release (uname -r) — set by build system */
/* UTS_VERSION — kernel version (uname -v) — set by build system */

/* UTS_MACHINE — architecture */
#define UTS_MACHINE "x86_64"

/* UTS_DOMAINNAME — NIS domain */
#define UTS_DOMAINNAME "(none)"
```

The `UTS_RELEASE` and `UTS_VERSION` are auto-generated by the build system
from `KERNELRELEASE` and the build timestamp. You don't need to edit `uts.h`
directly — controlling `LOCALVERSION`, `KBUILD_BUILD_USER`, and
`KBUILD_BUILD_TIMESTAMP` is sufficient.

## 6. Verification

After building and booting:

```bash
# Release string (uname -r)
uname -r
# → 6.19.6-Hyperion-0.1.0

# Full version string (uname -v)
uname -v
# → #1 SMP PREEMPT Linux 6.19.6-Hyperion-0.1.0 (Soumalya Das) 2026

# All info
uname -a
# → Linux hyperion 6.19.6-Hyperion-0.1.0 #1 SMP PREEMPT ... (Soumalya Das) 2026 x86_64 GNU/Linux

# /proc interface
cat /proc/version
# → Linux version 6.19.6-Hyperion-0.1.0 (Soumalya Das) (gcc version 13.2.0 ...) #1 SMP PREEMPT 2026

# Kernel banner in dmesg
dmesg | head -3
# → [    0.000000] Linux version 6.19.6-Hyperion-0.1.0 (Soumalya Das) ...
```

## Summary of Changes

| File / Method | What it controls | Required? |
|---|---|---|
| `Makefile` `EXTRAVERSION` | Base version suffix | Optional (use CONFIG_LOCALVERSION instead) |
| `CONFIG_LOCALVERSION` | Final version suffix appended | ✅ Yes |
| `CONFIG_LOCALVERSION_AUTO=n` | Prevents git hash overriding version | ✅ Yes |
| `KBUILD_BUILD_USER` env var | Author name in uname -v | ✅ Yes (set in build script) |
| `KBUILD_BUILD_HOST` env var | Host name in version string | Recommended |
| `KBUILD_BUILD_TIMESTAMP` env var | Build timestamp | Recommended |
| `CONFIG_DEFAULT_HOSTNAME` | Default system hostname | ✅ Yes ("hyperion") |
| `init/version.c` | Direct source edit | Optional (env vars preferred) |
